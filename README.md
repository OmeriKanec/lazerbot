В этом репозитории находится учебный проект по робототехнике.  
Это робот с 4 колёсами, управляемыми OmniWheelDriveController, с манипулятором, управляемым moveit, камерой, тепловой камерой и лидаром. Также используется nav2 для навигации.  
Робот создан на ros2 jazzy и сопутствующим стеке программ.

Команды:
1. Сборка:  
colcon build  
source install/setup.bash  
3. Запуск симуляции мира с роботом и rviz с видом с камеры:  
ros2 launch lazerbot_launch lazerbot.launch.py  
4.1 Запуск навигации со slam:  
ros2 launch lazerbot_launch navigation_with_slam.launch.py  
4.2 Запуск навигации:  
ros2 launch lazerbot_launch navigation.launch.py  
4.3 Запуск маппинга:  
ros2 launch lazerbot_launch mapping.launch.py  
(Запускать только 1 пункт из 4.1-4.3, только после запуска мира)  
5. Запуск управления манипулятором(только после запуска мира):  
ros2 launch lazerbot_launch start_moveit.launch.py  

Краткое описание структуры проекта:  
В lazerbot_control в папке config находятся yaml конфиги для моста, навигации, а так же ros контроллеров.  
В lazerbot_description находятся папки urdf, в которой лежат файлы описания робота, meshes, в которой лежат модели для робота, maps, где лежат карты для навигации, config, где лежат rviz файлы, и worlds, где лежат файлы газебо миров.  
В lazerbot_launch находится папка launch, со всеми launch файлами написанными для этого проекта.  
В lazerbot_moveit_config находятся файлы автоматически сгенированные moveit setup assistant, часть была изменена, так как иначе не работает(в moveit_controllers добавлены action_ns: follow_joint_trajectory default: true, в joint_limits.yaml указаны лимиты).  

Трудности при создании проекта:  
Множество раз были проблемы с взаимодействием разпных частей между собой из-за неправильных названий топиков или типов сообщений присылаемых туда.   
Например: навигиция посылала на контроллер Twist, а в контроллере почему то не сработал парметр use_stamped_vel: false, поэтому оказалось проще прокинуть через всю навигацию этот же параметр, но с true.  
Также некоторе время не замечал, что навигация подписывается на /odom, а контроллер публикует в /omni_wheel_drive_controller/odom.  
Было и множество других трудностей, все описывать не буду, одни из последних, с тем, что робот поворачивался во время езды по прямой удалось в основном решить с помощью добавления 4 колеса.
